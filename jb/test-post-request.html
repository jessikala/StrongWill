❓ Troubleshooting

❌ Request sent but no row in Google Sheets
Check Apps Script Executions:
  1. Open Apps Script editor
  2. Click "Executions" (clock icon) on left sidebar
  3. Look for recent executions
  4. If you see errors, click to view details
  5. Common error: "Cannot read property 'postData'" = Wrong deployment type
Solution:

Create a NEW deployment as "Web app" (not API executable)

❌ Failed to fetch / CORS error
This is normal for POST requests!

Due to CORS restrictions, POST requests return an "opaque" response. This doesn't mean it failed - check Google Sheets directly.

What to check:
  • Open your Google Sheet and refresh
  • Check Apps Script Executions log
  • If execution ran successfully, the data was saved
❌ No executions showing in Apps Script
Request isn't reaching your script

Possible causes:

  • Wrong URL
  • Deployment not set to "Anyone"
  • Deployment is inactive/archived
  • Old deployment URL (create NEW deployment)
